# Cyber Wingman Marketing Homepage UI Migration Design

## Purpose
Integrate the UI frontend code generated by Kimi (located in `Kimi_Agent_欧美两性AI UI优化`) into the existing `Cyber_Wingman/frontend` Next.js 15 application. 
The new UI will serve as the marketing homepage at the root route `/`, emphasizing the product's cyberpunk aesthetic and unique value propositions.

## Architecture & Integration Changes

### 1. Routing & Pages
*   **Move Chat:** The current chat interface at `src/app/page.tsx` will be relocated to a new route: `src/app/chat/page.tsx`.
*   **New Homepage:** A new `src/app/page.tsx` will be created using the structure from Kimi's `App.tsx` (containing Hero, Features, Testimonials, etc.).
*   **Navigation Links:** All primary call-to-action buttons (e.g., "GET STARTED", "START FREE TRIAL") on the new homepage will route users to `/chat`.

### 2. Component Migration
*   **Sections:** The Kimi `sections/` directory components (e.g., `Hero.tsx`, `FeaturesSection.tsx`, `Navbar.tsx`, etc.) will be migrated directly into `src/components/sections/` in the frontend app.
*   **UI Components:** The custom Kimi components from `components/` (e.g., `CharacterCard.tsx`, `FeatureBlock.tsx`, `GlowButton.tsx`, `ScrollReveal.tsx`) will be moved into the existing `src/components/` directory.
*   **Assets:** The images from Kimi's `public/images/` will be copied to the current frontend's `public/images/`.

### 3. Styling & Configuration
*   **Tailwind Config:** The color tokens (e.g., `cyber-black`, `cyber-pink`), custom fonts (`Inter`, `JetBrains Mono`), and animations (e.g., `pulse-glow`, `scanline`) from Kimi's `tailwind.config.ts` will be merged into the existing `src/app/globals.css` using CSS variables or inside the `tailwind.config.ts` depending on the current tailwind v4 setup. (Note: Current frontend uses Tailwind v4 with CSS variables).
*   **Dependencies:** Both projects already utilize `framer-motion` and `lucide-react`. No new major dependencies need to be installed.

## Vercel Deployment Considerations
*   The approach relies purely on static components and standard Next.js client-side/server-side routing.
*   Moving the chat to `/chat` and creating a static `/` marketing page is highly SEO-friendly and fully compatible with Vercel's edge network.

## Success Criteria
*   The marketing homepage renders identically to the Kimi output, with all animations (`framer-motion`) and neon styling intact.
*   Clicking CTA buttons smoothly transitions the user to the `/chat` route where the existing chat application functions normally.
*   No regressions in the existing chat functionality.
